จงสวมบทบาทเป็น **Senior Full Stack Developer** ที่มีความเชี่ยวชาญในการทำ Codebase Analysis

**ภารกิจของคุณ (Your Mission):**
ทำการ Refactor และปรับปรุงฟีเจอร์ **"ระบบจัดการเทมเพลตสลิป (Slip Template Management)"** ให้เป็นแบบ **All-in-One** โดยที่คุณต้อง **"ค้นหาและตรวจสอบไฟล์ที่เกี่ยวข้องทั้งหมดด้วยตนเอง"** จากในโปรเจกต์ (ไม่ต้องรอให้ระบุ Path)

**ปัญหาที่ต้องแก้ไข:**
ปัจจุบันการตั้งค่าสลิปและการแสดงผลตัวอย่าง (Preview) ยังแยกส่วนกัน หรือยังใช้ข้อมูลจำลองแบบ Hardcode อยู่ ทำให้ Admin ไม่สามารถกำหนดข้อมูลตัวอย่าง (เช่น ชื่อผู้โอน, เลขบัญชี, ธนาคาร) ให้แตกต่างกันในแต่ละเทมเพลตได้

**ความต้องการของระบบ (Requirements):**

1.  **Codebase Discovery (การค้นหาอัตโนมัติ):**
    * ให้คุณสแกนหาไฟล์ที่เกี่ยวข้องกับ `SlipTemplate` ทั้งหมด (Schema, DTO, Service, Controller)
    * ค้นหาไฟล์ Frontend ของหน้าจัดการเทมเพลต (Admin Template Page) และ Component ที่ใช้แสดงตัวอย่างสลิป

2.  **Backend Upgrades:**
    * แก้ไข Database Schema เพื่อเพิ่ม Field สำหรับเก็บ "ข้อมูลจำลอง (Simulation Data)" ในระดับเทมเพลต เช่น `senderName`, `senderAccount`, `senderBankId`, `receiverName` ฯลฯ
    * อัปเดต API ให้รองรับการบันทึกและแก้ไขข้อมูลเหล่านี้

3.  **Frontend Consolidation:**
    * ปรับปรุงหน้าจัดการเทมเพลตของ Admin ให้มี Tab หรือ Section สำหรับตั้งค่า "ข้อมูลจำลอง" นี้
    * เพิ่มระบบเลือกธนาคาร (Bank Selector) โดยดึงข้อมูลจาก API ธนาคารที่มีอยู่จริงในระบบ
    * เชื่อมต่อข้อมูลจากฟอร์มเข้ากับส่วนแสดงตัวอย่าง (Live Preview) ทันที ห้ามใช้ Mock Data ที่เขียนฝังไว้

**คำสั่งการทำงาน:**
* วิเคราะห์โครงสร้างไฟล์เดิมก่อนเริ่มแก้
* เขียนโค้ดทับไฟล์เดิมให้สมบูรณ์ โดยให้ครอบคลุมทั้ง Backend และ Frontend
* ตรวจสอบว่า Type และ Interface ต่างๆ สอดคล้องกันทั้งระบบ

**Output:**
ขอโค้ดฉบับเต็มของไฟล์ที่คุณได้ทำการแก้ไข (Schema, Service, Page Component) พร้อมระบุว่าไฟล์นั้นคือไฟล์อะไรจากการที่คุณไปค้นหามา
