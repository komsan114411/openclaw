🤖 Virtual Channels คืออะไร?

Virtual Channel คือช่องทางเสมือนที่ให้บอทภายนอก (เช่น Thunder Solution บอทตรวจสลิป) เชื่อมต่อมาหาระบบของเราแทนที่จะเชื่อมตรง LINE

ประโยชน์: ควบคุมการสื่อสารทั้งหมด, ไม่ต้องแชร์ LINE credentials, ติดตามการใช้งานได้

Reply Message API:
POST {{ base_url }}/virtual-api/v2/bot/message/reply
Authorization: Bearer {{ channel.access_token }}

Content API:
GET {{ base_url }}/virtual-api/v2/bot/message/{message_id}/content
Authorization: Bearer {{ channel.access_token }}

📝 คำแนะนำการใช้งานสำหรับ Thunder Solution

  1. ใช้ Channel ID, Channel Secret, และ Access Token ด้านบนในระบบของ Thunder
  2. ตั้งค่า Webhook URL เป็น: {{ base_url }}/virtual/{{ channel.channel_id }}/webhook
  3. ตั้งค่า API Base URL เป็น: {{ base_url }}/virtual-api/v2/bot
  4. ใช้ Access Token ในการเรียก API (Authorization: Bearer {{ channel.access_token }})
  5. Thunder จะได้รับ events จาก LINE ผ่านระบบเราและสามารถส่งข้อความกลับได้
