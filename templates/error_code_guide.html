เกี่ยวกับคู่มือนี้

คู่มือนี้รวบรวม Error Codes ที่พบบ่อยจาก LINE Messaging API พร้อมวิธีแก้ไขที่ชัดเจน เพื่อช่วยให้คุณสามารถแก้ไขปัญหาได้อย่างรวดเร็วและมีประสิทธิภาพ

ข้อผิดพลาดการยืนยันตัวตน (Authentication Errors)

401
Unauthorized - ไม่มีสิทธิ์เข้าถึง

เกิดขึ้นเมื่อ Channel Access Token ไม่ถูกต้องหรือหมดอายุ ระบบไม่สามารถยืนยันตัวตนของคุณได้

วิธีแก้ไข:
  • ตรวจสอบว่า Channel Access Token ถูกต้อง
  • ตรวจสอบว่า Token ยังไม่หมดอายุ
  • ลองสร้าง Token ใหม่จาก LINE Developers Console
  • ตรวจสอบว่าใช้ Token ของ Channel ที่ถูกต้อง
403
Forbidden - ถูกปฏิเสธ

เกิดขึ้นเมื่อคุณพยายามเข้าถึงทรัพยากรที่ไม่มีสิทธิ์ หรือ Channel ของคุณถูกระงับ

วิธีแก้ไข:
  • ตรวจสอบสถานะของ LINE Official Account
  • ตรวจสอบว่า Channel ไม่ถูกระงับ
  • ตรวจสอบ Permissions ของ Channel
  • ติดต่อ LINE Support หากปัญหายังคงอยู่

ข้อผิดพลาดคำขอ (Request Errors)

400
Bad Request - คำขอไม่ถูกต้อง

เกิดขึ้นเมื่อข้อมูลที่ส่งไปไม่ถูกต้องตามรูปแบบที่ LINE API กำหนด

วิธีแก้ไข:
  • ตรวจสอบรูปแบบ JSON ที่ส่งไป
  • ตรวจสอบว่า required fields ครบถ้วน
  • ตรวจสอบ data types ของแต่ละ field
  • ดู error message เพื่อหาสาเหตุที่แน่ชัด
429
Too Many Requests - ส่งคำขอมากเกินไป

เกิดขึ้นเมื่อส่ง API requests มากเกินกว่า rate limit ที่ LINE กำหนด

วิธีแก้ไข:
  • ลดความถี่ในการส่ง requests
  • ใช้ batch API สำหรับส่งข้อความหลายคน
  • เพิ่ม delay ระหว่างการส่ง requests
  • ตรวจสอบ rate limits ของ LINE API

ข้อผิดพลาดเซิร์ฟเวอร์ (Server Errors)

500
Internal Server Error - เซิร์ฟเวอร์ขัดข้อง

เกิดขึ้นเมื่อเซิร์ฟเวอร์ของ LINE มีปัญหา ไม่ใช่ความผิดของคุณ

วิธีแก้ไข:
  • รอสักครู่แล้วลองใหม่อีกครั้ง
  • ตรวจสอบ LINE Developers Status Page
  • ใช้ retry mechanism ในโค้ด
  • ติดต่อ LINE Support หากปัญหายาวนาน
503
Service Unavailable - บริการไม่พร้อมใช้งาน

เกิดขึ้นเมื่อบริการของ LINE กำลังอยู่ในช่วง maintenance หรือมีปัญหาชั่วคราว

วิธีแก้ไข:
  • รอจนกว่าบริการจะกลับมาปกติ
  • ตรวจสอบประกาศจาก LINE
  • ใช้ queue system เพื่อ retry ภายหลัง
  • แจ้งผู้ใช้ว่าระบบกำลังมีปัญหาชั่วคราว

ข้อผิดพลาดการส่งข้อความ (Message Errors)

404
Not Found - ไม่พบผู้ใช้

เกิดขึ้นเมื่อพยายามส่งข้อความไปยัง User ID ที่ไม่มีอยู่หรือยังไม่ได้เป็นเพื่อนกับ Bot

วิธีแก้ไข:
  • ตรวจสอบว่า User ID ถูกต้อง
  • ตรวจสอบว่าผู้ใช้ยังเป็นเพื่อนกับ Bot อยู่
  • ผู้ใช้อาจบล็อก Bot แล้ว
  • ลบ User ID ที่ไม่ใช้งานออกจากระบบ

เคล็ดลับในการดีบัก

  • เปิดใช้งาน logging เพื่อดู error messages โดยละเอียด
  • ใช้ LINE Messaging API Simulator เพื่อทดสอบ
  • อ่าน official documentation ของ LINE อย่างละเอียด
  • เก็บ log ของ API responses เพื่อวิเคราะห์ปัญหา
  • ใช้ try-catch เพื่อจัดการ errors อย่างเหมาะสม
  • ติดตาม LINE Developers Blog สำหรับข้อมูลอัปเดต
